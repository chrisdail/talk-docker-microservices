What are Microservices?

“In computing, microservices is a software architecture style, in which complex applications are composed of small, independent processes communicating with each other using language-agnostic APIs. These services are small, highly decoupled and focus on doing a small task.” - Wikipedia

Benefits

  • Smaller codebase, easier to understand
  • Easier to scale independent services
  • Easier to upgrade, version or replace a service
  • Faster build/deploy
  • Choice of technology stack

Downside

Can create messy deployment

Enter Docker

Docker

  • Process Isolation using "Containers"
  • High-level API/CLI to manage Containers
  • Built on Linux using cgroups, namespaces
  • Run multiple processes on single machine with separate environments

It's a Virtual Machine

I know this!

Docker Concepts

  • Docker uses Union File System
  • "Layers" are multiple filesystems on top of each other
  • "Images" are portable, read-only layers
  • Containers run in a writable filesystem on top of an image

Docker Containers

  • Running process on top of an image
  • Can run multiple containers off the same image

Dockerfile

  • Instructions file to docker build
  • Creates an image based on another image
# Firefox over VNC
FROM ubuntu

# Install vnc, xvfb in order to create a 'fake' display and firefox
RUN apt-get update && apt-get install -y x11vnc xvfb firefox
RUN mkdir ~/.vnc

# Setup a password
RUN x11vnc -storepasswd 1234 ~/.vnc/passwd

# Autostart firefox (might not be the best way, but it does the trick)
RUN bash -c 'echo "firefox" >> /.bashrc'

EXPOSE 5900
CMD    ["x11vnc", "-forever", "-usepw", "-create"]

Docker Registry

  • Central repository for exchanging images
  • Docker can push and pull from a registry
  • Public Images on Docker Hub Registry
  • Private Registry

The Good

  • Isolation of services, but more lightweight than VMs
  • Near bare-metal performance, start as fast as processes
  • Simple deployment, upgrade
  • No library compatibility problems
  • Aligns nicely with Microservices

The Bad

  • Security
  • Patching for all containers (think Shellshock)
  • Can be more difficult to debug
  • Docker is still very new and changing rapidly
